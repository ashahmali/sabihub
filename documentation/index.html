Introduction

Apex Responsive HTML Theme offers a unique intuitive interface, featuring the trendy one page layout design, and 6 built in colour themes. You can also easily customize colours and styling with valid and optimized CSS3. Built on Bootstrap 3, Apex has been carefully crafted to offer a reliable and robust theme for your project.

Also included is a separate blog section, which has a blog archive, single blog post pages and supports 4 different blog post types - text, single image post, multiple image post and video post.

Hero Element

The Hero element is the first thing you see on your website. By default it is only displayed on the index.html file, but it could be used on any of your other pages as well.

The Hero Element should be the first element placed within the <body> tag of your page.
We suggest not using both a Hero & Header element on the same page.

Header Element

The Header element is the first thing you see on your website. By default it is only displayed on the sub-pages of the site, but it could also be used on the index.html page instead of the Hero element.

The Header element uses many of the same classes as the Sections to keep styling consistent.

The Header Element should be the first element placed within the <body> tag of your page.
We suggest not using both a Hero & Header element on the same page.

Example Header Element:

Menus & Site Navigation

There are 2 types of menus for Apex: The Hero menu and Primary Menu. Both menus can be setup to scroll down to a selected section ID or link to another page.

You must add unique ID's to your sections in order to scroll or link to that section.

Section ID Example:

<section id="about" class="apex-section">

ID Link Example:

<li><a href="#about"><i class="apex-icon-gear"></i>About</a></li>

External Link Example:

<li><a href="blog.html"><i class="apex-icon-write"></i>Blog</a></li>

To scroll to the top when clicking the log set the link to "/":

<a id="logo" href="/">
  <img src="assets/images/logo.png" alt="Apex" width="75" height="21">
</a>

To enable page scrolling, add "apex-menu-scroll" to the body element:

<body class="apex-menu-scroll">

To autohide the primary menu when a page is scrolled up past a Hero or Header element, add "apex-hide-nav" to the body element:

<body class="apex-hide-nav">

Contact Form

Apex comes with a working php and ajax email sending form, all you have to do to start receiving emails is to open the /assets/php/commentform.php file and edit lines 6, 7, and 12 (optional). These lines specify your own email address, the subject of the emails, and an optional logo to include in the emails.

If you have a properly configured mail server on your hosting, you shouldn't have to do anything else.
 
// Type in your email address to receive the mail
$to = "youremail@gmail.com";
$subject = "Sample message from Apex";

// If you wish to show a logo in the mail, paste the URL here.
// For example: http://mywebsite.com/mylogo.png
// Remember http://
$logo_url = "";

$('.mtphr-rotator').mtphr_rotator({

  rotate_speed : 10, // Rotation speed, in hundredths of a second
  rotate_ease : 'easeOutExpo', // Options: See options here http://gsgd.co.uk/sandbox/jquery/easing/
  nav_reverse : 0, // Options: 0 or 1. Reverse the slide direction when going backwards
  event_prefix : 'mtphr_rotator', // Prefix used for event labels
  resource_container_class : '.mtphr-rotator-resource-container', // Specify the container class
  resource_class : '.mtphr-rotator-resource', // Specify the resource class
  nav_prev_class : '.mtphr-rotator-nav-prev', // Specify the previous button class
  nav_next_class : '.mtphr-rotator-nav-next', // Specify the next button class
  nav_controls_class : '.mtphr-rotator-navigation', // Specify the navigation controls class
  before_change : function( $rotator_container, $rotator ){}, // Before change event
  after_change : function( $rotator_container, $rotator ){}, // After change event
  after_load : function( $rotator_container, $rotator ){} // After load event
});

1. Each Gallery Block needs to have a data-gallery="path-to-gallery-rotator-here" attribute that tells jQuery which gallery file to load when the block is clicked.

2. Each link within the mtphr-gallery-filter section needs to have a data-filter=".class-name-here" attribute. These will filter the blocks that contain that particular class in their class attribute.

Basic Gallery Rotator HTML Structure:

Please view one of the files located in the "galleries" directory.

Using the Load More Button:

Add a path to another file that contains more Gallery Blocks to load in via ajax. The sample file for this is located at "galleries/galleries-more.html"

Include the CSS in your HTML <head>:

<link rel="stylesheet" href="assets/mtphr-gallery/mtphr-gallery.css" type="text/css" media="all">

Include the jQuery classes before the closing HTML </body> tag:

<script type="text/javascript" src="assets/js/jquery.isotope.min.js"></script>
<script type="text/javascript" src="assets/js/jquery.isotope.perfectmasonry.js"></script>
<script type="text/javascript" src="assets/mtphr-gallery/mtphr-gallery.js"></script>

Initiate your galleries after the included jQuery class before the closing HTML </body> tag:

$('.mtphr-gallery').mtphr_gallery();

Parallax Backgrounds

Apex uses the Metaphor Parallax plugin to create parallax backgrounds on sections throughout the site.

Add a background to your sections in the assets/css/custom/css file:

#quotes {
  background-image: url(../images/Parallax1.jpg);
}

Add a data-mtphr-parallax-speed attribute to your section:

<section id="client-quotes" class="apex-section apex-style-dark apex-style-wide" data-mtphr-parallax-speed="6">
The speed can range anywhere from 1 and up. The higher the number the slower the background will scroll up along with the page.

Include the jQuery class before the closing HTML </body> tag:

<script type="text/javascript" src="assets/mtphr-parallax/mtphr-parallax.js"></script>

Initiate the parallax background after the included jQuery class before the closing HTML </body> tag:

$('html.no-touch #quotes').mtphr_parallax();
I have added html.no-touch to the selectors to disable the parallax on touch devices. You do not have to, but the parallax ends up being very jittery on touch devices.

WOW.js Animations

Apex uses WOW.js http://mynameismatthieu.com/WOW/index.html, along with Animate.css http://daneden.github.io/animate.css/ to animate elements when they become veiwable.

Please view the links above for full documentation on how to use these plugins.

Include the CSS in your HTML <head>:

<link rel="stylesheet" href="assets/css/animate.min.css" type="text/css" media="all">

Include the jQuery class before the closing HTML </body> tag:

<script type="text/javascript" src="assets/js/wow.min.js"></script>

Initiate your sliders after the included jQuery class before the closing HTML </body> tag:

var wow = new WOW({
  offset: 150,
  mobile: false
});
wow.init();
